Device Drivers

Description

Here provides a common device driver architecture

  • Sources are centralized at \sdk\driver\itp
  • Controled by unify function interface including open/close/read/write/lseek/ioctl
  • Easy to add a new device driver

Add a new device driver

  • Modify \sdk\include\itp.h
    1. find ITPDeviceType enumeration, add id of the device driver like:
      ...
      ITP_DEVICE_ETHERNET = 29,
      ITP_DEVICE_WIFI = 30,
      ITP_DEVICE_NEW_DEVICE = 31,
      ITP_DEVICE_MAX
      } ITPDeviceType;
      <li>find IOCTL section, add any new ioctl id if necessary:
      ...
      #define ITP_IOCTL_IS_AVAIL 32
      #define ITP_IOCTL_GET_INFO 33
      #define ITP_IOCTL_NEW_CMD 34
      <li>find device entry section, add the new device entry:
      ...
      // Ethernet device
      extern const ITPDevice itpDeviceEthernet;
      // WiFi device
      extern const ITPDevice itpDeviceWifi;
      // New device
      extern const ITPDevice itpDeviceNew;
  • Create the new device driver
    1. Create an itp_<device>.c at \sdk\driver\itp\ by copying other itp_<device>.c
    2. Modify the device entry like below.
      const ITPDevice itpDeviceNewDevice =
      {
      ":newdevice", // open("newdevice")
      itpOpenDefault, // open()
      itpCloseDefault, // close()
      NewDeviceRead, // read()
      itpWriteDefault, // write()
      itpLseekDefault, // lseek()
      NewDeviceIoctl, // ioctl()
      NULL // *info
      };
      where the "newdevice" is the device name used by the first parameter of open() Overwrite these default functions if necessary.
  • Add configuration to configuration menu
    1. Edit \sdk\Kconfig
    2. find Peripheral section, add the new device configuation like:
      ...
      config NEW_DEVICE_ENABLE
      bool "New Device Enable"
      default y
      <li>Add GPIO configuration if necessary. Find GPIO section, add the new GPIO configuation like:
      ...
      config GPIO_NEW_DEVICE
      string "New Device Pin"
      depends on NEW_DEVICE_ENABLE
      default "30"
      help
      New Device Pin
  • Add compiler options if necessary. Edit \sdk\def.cmake:
    1. Find Peripheral section, add the new compiler option like:
      ...
      if (DEFINED CFG_NEW_DEVICE_ENABLE)
      add_definitions(
      -DCFG_NEW_DEVICE_ENABLE
      )
      endif()
      <li>Find GPIO section, add the new compiler option if necessary:
      ...
      if (DEFINED CFG_GPIO_NEW_DEVICE)
      add_definitions(
      -DCFG_GPIO_NEW_DEVICE
      )
      endif()
  • Link this device driver library
    1. Edit \sdk\build.cmake, add code as below:
      if (DEFINED CFG_BUILD_NEW_DEVICE)
      target_link_libraries(${CMAKE_PROJECT_NAME}
      new_device
      )
      endif()
      Notice the link order, if this deive driver uses other libraries, these link commands of libraries must be after this link command of this new device driver like:
      ...
      if (DEFINED CFG_BUILD_NEW_DEVICE)
      target_link_libraries(${CMAKE_PROJECT_NAME}
      new_device
      )
      endif()
      if (DEFINED CFG_BUILD_OTHER_DEVICE)
      target_link_libraries(${CMAKE_PROJECT_NAME}
      other_device_used_by_new_device
      )
      endif()
      ...
  • Add this device driver to build tree
    1. Edit \project\<project>\Kconfig, add code as below:
      ...
      config NEW_DEVICE_ENABLE
      def_bool y
      source "$CMAKE_SOURCE_DIR/sdk/Kconfig"
